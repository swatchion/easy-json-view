# EazyJsonView 需求规范文档

## 1. 项目概述

### 1.1 应用信息
- **应用名称**: EazyJsonView
- **版本**: v1.0.0
- **框架**: Dioxus 0.6.3
- **目标平台**: 桌面版 GUI（当前专注于 Linux Ubuntu 22.04）
- **主要功能**: JSON 字符串的美化、格式化和历史记录管理

### 1.2 项目目标
开发一个简洁高效的桌面版 JSON 格式化工具，提供直观的用户界面和便捷的历史记录管理功能。

## 2. 技术栈规范

### 2.1 核心技术
- **框架**: Dioxus 0.6.3（最新稳定版）
- **渲染器**: Desktop 渲染器（支持多渲染器架构，未来可扩展 Web 等）
- **包管理**: Cargo（Rust 官方工具）
- **样式**: Tailwind CSS 4
- **UI 组件**: Dioxus 标准组件库

### 2.2 依赖库
- **JSON 处理**: `serde_json` - JSON 解析和格式化
- **配置文件**: `serde_toml` - TOML 配置文件处理
- **数据库**: `sqlx` - SQLite 数据库操作（支持异步和迁移）
- **错误处理**: `anyhow` - 人性化错误处理
- **其他**: 根据开发需要添加

### 2.3 构建和部署
- **目标平台**: Linux Ubuntu 22.04
- **打包方式**: AppImage 格式
- **分发**: 单文件可执行程序

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 JSON 格式化
- **输入**: 支持用户在下层文本区域粘贴 JSON 字符串
- **触发**: 回车键或提交按钮触发格式化
- **输出**: 在中层区域显示格式化后的 JSON
- **验证**: 无效 JSON 时使用 Modal 提示用户格式错误

#### 3.1.2 格式化选项（当前版本）
- **缩进宽度**: 可配置缩进空格数量（2、4、8 空格选项）
- **其他选项**: 作为禁用状态的表单元素预留

#### 3.1.3 历史记录管理
- **自动保存**: 每次成功格式化后自动保存到历史记录
- **记录标识**: 默认使用时间戳作为记录名称和 ID
- **删除功能**: 支持删除单条历史记录
- **无限制**: 历史记录数量不做限制

### 3.2 用户界面

#### 3.2.1 窗口规格
- **默认大小**: 1200x800 像素
- **最小大小**: 800x600 像素
- **可调整**: 支持窗口大小调整和最大化

#### 3.2.2 布局结构

**左侧边栏（宽度占比 25%）**
1. **搜索区域**（顶部）
   - 搜索输入框
   - 支持按名称、内容过滤历史记录

2. **历史记录列表**（中部）
   - 显示历史记录列表
   - 每项显示记录名称（时间戳）
   - 铅笔编辑按钮（预留，当前版本禁用）

3. **应用配置**（底部）
   - 齿轮图标按钮（预留入口）

**右侧主区域（宽度占比 75%）**
1. **上层**（高度占比 15%）- JSON 格式化选项
2. **中层**（高度占比 50%）- 格式化输出显示
3. **下层**（高度占比 35%）- JSON 输入文本区域

### 3.3 用户交互流程

1. **启动应用** → 检查数据目录 → 初始化界面
2. **输入 JSON** → 在下层文本区域粘贴内容
3. **触发格式化** → 回车或点击按钮
4. **验证处理** → 有效则格式化，无效则 Modal 提示
5. **显示结果** → 中层显示格式化后的 JSON
6. **保存历史** → 自动添加到左侧历史记录列表顶部
7. **查看历史** → 点击历史记录项查看之前的内容

## 4. 数据存储

### 4.1 存储位置
- **数据目录**: `~/.config/EazyJsonView`
- **配置文件**: `config.toml`
- **数据库文件**: `history.db`

### 4.2 数据库结构

```sql
-- 历史记录表
CREATE TABLE history_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,           -- 记录名称（默认时间戳）
    content TEXT NOT NULL,        -- 原始 JSON 内容
    formatted_content TEXT NOT NULL, -- 格式化后的内容
    note TEXT,                    -- 备注（预留）
    tags TEXT,                    -- 标签（预留，JSON 格式）
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 配置表
CREATE TABLE app_config (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 4.3 配置文件结构

```toml
[app]
version = "1.0.0"
window_width = 1200
window_height = 800

[formatting]
indent_size = 2
# 预留其他格式化选项

[ui]
# 预留 UI 相关配置
```

## 6. 非功能性需求

### 6.1 性能要求
- **启动时间**: < 2 秒
- **格式化响应**: < 500ms（中等大小 JSON）
- **内存使用**: < 100MB（正常使用）

### 6.2 兼容性
- **操作系统**: Linux Ubuntu 22.04+
- **架构**: x86_64
- **依赖**: 最小化系统依赖

### 6.3 可用性
- **界面响应**: 流畅的用户交互
- **错误提示**: 清晰的错误信息
- **数据安全**: 本地数据存储，无网络依赖
